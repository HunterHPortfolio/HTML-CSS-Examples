APUT Base Test 1.1.5 - Association Processing
IEEE Std 802.11™-2012 Edition, Subclauses 10.3, 4.5, and 9.3.2.2

Test 1.1.5 Part a

The APUT should respond to MSDU3-MSDU6 (MSDU3 on frame 289)
with an Association Response (frame 291).
A. If the APUT sends an Association Response with a status code indicating it is unsuccessful, then it should not be able to pass class 3 traffic. Further in the trace there is an ICMP Echo Request, if the APUT Replies on an unsuccessful Association - then it would be a fail.
B. If the APUT sends an Association Response indicating it's successful, then there should be a successful ICMP Echo Request/Reply exchange afterwards

APUT Base Test 1.2.4 - Directed Fragmentation
IEEE Std 802.11™-2012 Edition, Subclauses 9.2.7, 9.3.2.6, and 9.5

Test 1.2.4 Part a

Dont let this one scare you
We will be looking into the Fragmented frames for this case (Frame 196, 198, and 200),
along with the final fragment/ICMP Echo Reply frame (frame 202).
1. first we check the size - the fragmented frames (196, 198, and 200) should be 512-bytes
2. we need to check that the fragments correctly set the more fragments bit - the fragmented frames (196, 198, and 200) should be set to 1 while the reply frame (202) should be set to 0.
3. The destination address can be found on the trace - it should normally be Xerox_00:00:01 in all cases for fragments and 192.168.1.101 for the reply frame.
4. For the sequence number, they should all be the same number.
5. For Fragment numbers, the first fragment should start at 0 and then increment by 1.

Test 1.2.4 Part b

The only difference from this part and 1.2.4 Part a - is the size of the fragments. They should all be set to 300-bytes instead.
It should correctly set the More Fragments bit.
Correctly set the Destination.
Maintain the same sequence number for all fragments of each MSDU.
Finally, begin the fragments at 0 and increment by 1.

APUT Base Test 1.2.5 - Multirate Support
IEEE Std 802.11™-2012 Edition, Subclause 9.7

Test 1.2.5 Parts a-f

This test sounds complicated but is pretty simple, we are just checking that the APUT transmits Control response frames (CTS and ACK frames) at one of the basic rates set in the BSSBasicRateSet at a rate less than or equal to the rate of the previous frame sent.
For Part a-c
The Basic Rates = 1 and 2
Extended Rates = 5.5 and 11
For Parts d-f
Basic Rates = 6, 12, and 24
Extended Rates = 9, 18, 36, 48, and 54
As you see here, this is part d so we are looking to see if the Data Rate matches any of the rates on the lower table and it does, this is a Pass!
