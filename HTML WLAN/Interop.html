Interoperability Testing

IOL Interop Ping Tool



There isn't all too much to determine a pass or fail within the Interoperability test suite:
In all cases, verify that the STA and AP connect properly and the STA responds with ICMP responses and the AP passes ICMP echo requests and responses without error and with a ping loss percentage no greater than 5%.
We have a tool that is very helpful in determining this

IOL Interop Splash Screen

Once you launch the Interop ping tool, select either APUT or STAUT

Configuration/Test Screen

There's a few things we need to do here to Configure everything. The STA1 IP address is the DUT IP address. STA2 IP Address is only used for test 1.5 (Which would be the IP of the Legacy device you are using to test). Test number doesn't matter too much for tests 1.1-1.4. The Ping section in the very top right is very useful for troubleshooting. You can either choose to send ping requests to STA1, STA2 or both to see which is responding (or not). The Continuous box sends continuous pings, this can be used if you are configuring a device that is not responding with replies. Same with the slow option, this can be used to send pings in reduced time for help with troubleshooting.

Running The Test

Once set up correctly, run a test run, you should see ping replies cycling through.You can see the amount of time each ping takes to reach the TS to the right.

Results

This is displayed on the right hand side, it will tell you if it Passed or Failed.
a Fail constitutes ping loss greater than 5%.
