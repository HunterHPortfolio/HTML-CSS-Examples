Group 1 - Field Checking

WPA Test 1.1.1 - TKIP Countermeasures
IEEE Std. 802.11-2007, Clause 11.4.2.4 and 11.4.2.3 and WPA for 802.11 version 2.0, Appendix H, Figure 8

Test 1.1.1 Part a

The APUT Should respond to every transmission of MSDU1 (frame 164)
with an Acknowledgement (Frame 165)
followed by an ICMP Echo Response (Frame 166)

Test 1.1.1 Part b

The APUT Should Acknowledge both MPDU2's (frames 189 and 202)followed by not responding with an ICMP Echo Response
The APUT Should Disassociate or Deauthenticate in response to the second MPDU2 (Frame 204) with a reason code indicating MIC failure

Test 1.1.1 Part c

The APUT Should Acknowledge both MPDU3's (frames 153 and 263). followed by not responding with an ICMP Echo Response
The APUT Should then Disassociate or Deauthenticate in response to the second MSDU3 (Frame 165)

Test 1.1.1 Part d

The APUT Should not respond to MSDU4 (frame 164) at all.

Test 1.1.1 Part e

The APUT Should Acknowledge both MPDU5's.
The APUT Should Not Disassociate or Deauthenticate.

Test 1.1.1 Part f

The APUT Should Acknowledge both transmissions of MPDU6 (frame 144)
and MPDU3 (frame 157).
Upon reception of MPDU3 the APUT should Disassociate or Deauthenticate. (Frame 159)


WPA Test 1.1.2 - TKIP Replay Protection
IEEE Std. 802.11-2007, Clause 11.4.2.4 and 11.4.2.3 and WPA for 802.11 version 2.0, Appendix H, Figure 8

Test 1.1.2 Part a

This is a failure case
The APUT should transmit the first group key with a TSC of 1 or 0.
you can find this number in the "TKIP Ext. Initialization Vector" field under IEEE 802.11 Data.
As you see here, the TSC is 2, it should be 1 or 0.

Test 1.1.2 Part b

The APUT should use incrementing TSCs for each ICMP Echo Response transmitted.

Test 1.1.2 Part c

This is similiar to 1.1.2 Part a
The APUT should transmit the first Group Key with a TSC of 1 or 0,
The Group Key is represented as a Data frame on this trace.
The APUT should also increment the TSCs for each transient key and Group Transient Key.

Test 1.1.2 Part d

The APUT should acknowledge MSDU4 followed by discarding it.
The APUT should also reply to MSDU1 (Arp Request) with an ARP Response.

Test 1.1.2 Part e

The APUT should forward an ICMP Echo Response to MSDU5-11.
The APUT should then discard MSDU12 after transmitting an ACK for it.

Test 1.1.2 Part f

This Test is Informational
The APUT may send a deauthentication or disassociation in response to the second transmition of MSDU4.
in this case the APUT processed the frame and did not deauthenticate or disassociate.
