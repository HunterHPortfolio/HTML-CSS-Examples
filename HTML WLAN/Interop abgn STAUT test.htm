BASIC SETUP

(Needs to be updated for tests 1.6, 1.7, and 1.8)

(1.6 and 1.7 has info written, 1.8 does not)

Physical Setup

1.       Set up 6 total AP’s ((case by case) if testing 2 devices, just need 3 AP’s (timing)) to power supply

2.       Connect all AP’s with Ethernet, if able (connected to Switch)

3.       Connect sniffer to switch via Ethernet

4.       Sometimes STAUT needs to be connected to laptop (bridged) in order to work (dev devices)

5.       Ensure your mobile device’s wireless is in airplane mode (to produce more accurate results in case mobile tries to be another entity during testing (good practice))

 

Folder Setup

1.       Find the Folders Template for the abgn Interop folders (sidebar in window explorer)

2.       Make sure the .txt for STAUT is included in directory

3.       Copy everything into correct vendor/device directory (or create new directory) (sidebar)

 

Program Setup

1.       Open OmniPeek, move it over to right monitor, make sure it’s on Sniffer profile

2.       Open .txt from the copied template in Notepad++, place in middle

3.       Open IOL Ping Tool (located on desktop), place far left

4.       Open CMD to run a basic ping to verify connected to device

A.      Command: ping -t (IP address of DUT), observe that it goes through

B.      If Ping doesn’t go through, you probably did something wrong (setup wise)

 

To Test Most Tests: (Pass/Fail: 1.1, 1.2, 1.3, 1.4, 1.9 and Informational: 1.10, 1.11)

(For 1.2, 1.5, 1.6, 1.7, 1.9 see “Test Specific” below for addition information)

1.       Configure all AP’s to the right settings (Security, Short Guard on/off, etc.), Make Sure It’s Saved

2.       Most of these tests can be configured through AP GUI

3.       Conduct Open Security Tests in Order: 1.1, 1.10, 1.11, 1.9 (see “Test Specific” for 1.9).

4.       Have 1 AP on at a time

5.       Start OmniPeek and observe 4-way handshake (Auth/Association/Keys/etc.)

6.       CMD Ping DUT IP to verify connected

7.       Stop OmniPeek

-----------------------------------------------------------------------------------------------------------------

1.       Ensure IOL Ping is on right test (1.1…etc.) + runs (5 runs if device is good, 10 if the device is meh)

2.       Start OmniPeek

3.       Start IOL ping tool

4.       Stop OmniPeek after 1st run finished (observe runs at bottom of Ping Tool)

5.       Save .pkt file to appropriate directory “APDevice_pass/fail.pkt”

6.       Let IOL ping tool run/finish

7.       Copy results into Notepad++ (Ping Tool Will Display results)

A.      Ping loss > 5% = FAIL

B.      Timeouts > 5   = FAIL

C.      If a test results in a FAIL, retest multiple times to see if it’s consistent

8.       Move to WPA2 Tests in Order: 1.1, 1.10, 1.11, 1.3, 1.4, 1.2 (see “Test Specific”), 1.9 (ran the same way as Open)

9.       Ensure that the right security type is configured: AES, TKIP, etc.

10.   Then WEP Tests: 1.1 Only

11.   Finally, WPA Tests: 1.1 Only

 

Test Specific

1.9: Failover

1.       Have 1 AP on (Make Sure all APs are switched back to default (20/40 MHz etc.))

2.       Have CMD open, run Ping -t (DUT IP) continuously

3.       Run OmniPeek continuously through all tests

4.       Have device associate to first AP

5.       Turn on 2nd AP

6.       Wait for 2nd AP to boot (look at destination address in OmniPeek for abundant changes to know)

7.       Shut off 1st AP

8.       Results are for 2nd AP, count how many pings are missed

9.       Go back to Step 1 until every AP has a result

10.   Stop OmniPeek, save as “device1_device2_device3_pass/fail.pkt” in order ran, when finished

11.   Only needs to be ran for Open and WPA2-PSK AES Security

 

1.2: Spatial Streams

 

1.       To determine how many spatial streams a device has, look at the Probe Request Frame

2.       Under MCS you will see specific data to determine how many streams the device has

3.       This test is ran just like all the other tests

Terms When Testing and Through This Guide

1.       Switch: A device that connects other devices to network (“hub”), series of Cat5 inputs

2.       AP: Access Point (router)

3.       n/s – not supported (when a DUT/AP doesn’t support that part of the test)

4.       Most Terms should be basic knowledge, if not, Google is your best friend!

 

Linux/Ubuntu Terminal Commands

(learn them - could be very helpful in future)

1.       Ifconfig : check network status

2.       Ls : list files in current directory

3.       Cd .. : up one directory

4.       Cd <flie> : go to that directory

5.       Ifconfig wlan0 <IP address> : set static IP

6.       Wpa_supplicant -dd -B -Dnl80211 -I wlan0 -c <config file> : connect device

7.       Killall wpa_supplicant : disconnect device
