RvT Setup

Non-Bridged Mode Instructions
(If the DUT can run an iPerf3 Server on its own)

  1. Connect a 4 Spatial Stream (4SS) AP from our Testbed to the lower chamber (This is normally our Netgear R7500)
  2. Replace each antanna on the 4SS Testbed AP with the wired antannae located in the lower chamber in order from left to right.
  3. Connect a networking (Cat-5) cable to any Ethernet port on the 4SS Testbed AP to the back of lower chamber
  4. Connect The Sniffer via ethernet to the lower chamber. The Ethernet port is located on the back of the octoBox
  5. The upper chamber houses the DUT, place it inside
  6. If the DUT cannot run an iPerf3 server then you would have utilize bridge mode (See "Bridged Mode Instructions Below).
    If the device can run an iPerf3 server - continue with these instructions.
  7. From the Sniffer PC - you can configure all of the settings for the 4SS Testbed AP (switching between 2.4GHz/5GHz basically)
  8. Have the DUT associate with the 4SS Testbed AP
  9. Run the iPerf3 server on the DUT (depends on DUT - see iPerf3 commands below)
  10. On the Sniffer PC, shift right click in the throughputscript directory (directory) and click "open command windo here).
  11. Type in "tclsh name_of_RvT_script" and execute
  12. Wait until done
  13. Rotate the DUT clockwise and run again



Bridged Mode Instructions
(If the DUT Can't run an iPerf3 Server)

  1. Follow steps 1-5 above
  2. For bridged mode, you need just another TS to act as the iPerf3 Server instead of the DUT
  3. This additional TS needs to be connected to the upper chamber via Ethernet
  4. The DUT needs to be connected via Ethernet as well (to the back of the upper chamber)
  5. Navigate the testbed ap to find the bridge mode settings (Advanced>Advanced Setup>Wireless Bridge)
  6. Enable bridge mode and set the IP to have a static IP
  7. Click on "Setup Bridge Mode Wireless Settings" - In here you can change between 2.4 and 5GHz as well as set the SSID and Password
  8. Click on Apply and then apply it again. Make sure you have the right Wireless radio switched on
  9. Continue with steps 10-13 above
  10. Note: If you are taking it out of Bridge Mode, make sure to set static IP, otherwise you might have to factory reset the AP and try again

Basic iPerf3 commands:

To start a server: "-s -p 5201"
To Start a Client: "-c [IP of Server] -p 5201"
To view additional parameters please see iperf documentation

Notes:

There is a file parser script so that the results can be displayed in a list rather then what iPerf displays. To run this all you have to do is move the script into the folder you are trying to parse, open CMD, type in tclsh into command prompt again and it will automatically parse it
