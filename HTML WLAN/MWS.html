Group 1 - Field Checking

WPA Test 1.1.1 - TKIP Countermeasures
IEEE Std. 802.11-2012, Clause 11.4.2.4, 11.4.2.3 and WPA for 802.11 Version 2.0, Clause 3.1, Appendix H, Figures 5 and 6

Test 1.1.1 Part a

The STAUT Should respond to each transmission of MSDU1 (frame 164)
with an Acknowledgement (Frame 165)
followed by an ICMP Echo Response frame (Frame 166)

Test 1.1.1 Part b

The STAUT Should discard each transmission of MPDU2 (ICMP Echo Requests Frame 570 and 589)
this should be followed by transmitting a disassociation or deauthentication frame
with a reason code indicating a MIC failure in response to the second transmission of MSDU2.
The STAUT should also transmit a disassociation or deauthentication frame upon reception of MSDU1.

Test 1.1.1 Part c

The STAUT Should discard each transmission of MPDU3 (frames 269 and 286)
this should be followed by transmitting a disassociation or deauthentication frame
with a reason code indicating a MIC failure in response to the second transmission of MSDU3.
The STAUT should also transmit a disassociation or deauthentication frame upon reception of MSDU1.

Test 1.1.1 Part d

The STAUT Should discard any transmissions of MSDU4 (frame 176).
The STAUT should also not send a disassociation or deauthentication frame upon reception of MSDU4.
The STAUT should then respond to MSDU1 with an Acknowledgement frame followed by an ICMP Echo Response

Test 1.1.1 Part e

The STAUT Should discard each transmission of MPDU5 (frame 194, represented as a data frame)
it should not be followed with a disassociation or deauthentication frame
The STAUT should also respond to MSDU1 with an Acknowledgement frame followed by an ICMP Echo Response.

Test 1.1.1 Part f

This is a Failure case
The STAUT Should discard both MPDU6 (frame 186)and MPDU3 (frame 198).
This should be followed by transmitting a disassociation or deauthentication frame
with a reason code indicating a MIC failure in response to the second transmission of MSDU3.
As you see here, the STAUT discards MSDU6 but responds to MSDU3
on top of not sending a Deauth or disassociation containing a reason code indicateing a MIC failure


WPA Test 1.1.2 - TKIP Replay Protection
IEEE Std. 802.11-20012, Clause 11.4.2.4, 11.4.2.3, and 11.4.2.1 and WPA for 802.11 version 2.0, Appendix H, Figures 5 and 6

Test 1.1.2 Part a

This is a failure case
the STAUT should transmit the first group key with a TSC of 1 or 0.
you can find this number in the "TKIP Ext. Initialization Vector" field under IEEE 802.11 Data.
The STAUT should also respond to transmissions of MSDU1-MSDU2 with incrementing TSCs (ARP Requests and ICMP Echo Requests).

Test 1.1.2 Part b

The STAUT should use incrementing TSCs for each response to the transmission of MSDU2 (ICMP Echo Request).

Test 1.1.2 Part c

This is similiar to 1.1.2 Part a
The STAUT should transmit the first Group Key with a TSC of 1 or 0,
The Group Key is represented as a Data frame on this trace.
The STAUT should also increment the TSCs for each transmission of MSDU1-MSDU2 (ARP Requests and ICMP Echo Requests).

Test 1.1.2 Part d

The STAUT should discard both transmissions of MSDU4 after Acknowledging both.
The STAUT should then respond to MSDU1 with an ARP Response.

Test 1.1.2 Part e

The STAUT should Acknowledge and forward an ICMP Echo Response to MSDU5-11.
The STAUT should then discard MSDU12.

Test 1.1.2 Part f

This Test is Informational
The STAUT may send a Deauthentication or Disassociation in response to the second transmition of MSDU2.
As you see here, the STAUT responded to MSDU2 and did not Deauthenticate of Disassociate.
